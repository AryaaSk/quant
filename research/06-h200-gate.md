# 06. H200 gate status (2026-05-15)

## Where the H200 fits

Phase 4 (transformer training) is the only stage that touches the H200. Phases 0-3 (bootstrap, screening, collection, structure, state) and Phase 5-6 (backtest, leaderboard) run locally on Mac with no GPU.

## Why the gate is not yet hit

GPU 6 is reserved for this project but is currently in use by Zoral continual-learning research. The plan was to do all local work first, then pause for explicit user confirmation that GPU 6 is free before any SSH.

In this iteration we did not need to wait: both markets that completed an end-to-end run (`_synthetic`, `betfair-eng-l1l2`) use `model: gbdt`, which trains on Mac CPU in seconds. So Phase 4 is currently unblocked-but-unused.

## What would hit the gate

Any market with `model: transformer` and all data sources implemented. Today those are zero (the transformer markets in the universe all depend on at least one stubbed adapter). To force a gate hit:

Option A: implement at least one stubbed adapter for an existing transformer market, e.g. `kalshi_contracts` so `kalshi-cpi` advances; then `bin/run-market kalshi-cpi` will halt at training and request `--confirm-gpu-free`.

Option B: edit `betfair-eng-l1l2.yaml` to set `model: transformer` and increase the state-vector dim back to 384. The collect / structure / state stages will succeed locally; `bin/run-market` will then halt at training and ask for `--confirm-gpu-free`. Run `bin/h200 --check` first to validate the SSH config offline.

## Sanity-checking the deploy helper without SSH

```bash
bin/h200 --check
```

Validates that the SSH key, host, user, remote dir, GPU index, and python path are all set sensibly. It does not attempt a connection.

## When GPU 6 frees up

```bash
H200_CONFIRMED=1 bin/h200 sync                                    # rsync the project
H200_CONFIRMED=1 bin/h200 run scripts/03_train.py <slug>         # train one market
bin/h200 pull runs/<slug>/                                       # bring checkpoints back
.venv/bin/python scripts/04_backtest.py <slug>                   # local walk-forward backtest
.venv/bin/python scripts/05_compare.py                           # update LEADERBOARD.md
```

## What lives where after the deploy

| location | content |
|---|---|
| Mac `~/Desktop/quant/` | source of truth |
| H200 `~/Aryaa/quant/` | mirrored project + training data + run outputs |
| Mac `runs/<slug>/<ts>/ckpt.pt` | pulled back after training |

The H200's `~/Aryaa/quant/` shares the `~/Aryaa/venv/` venv with the Zoral side so we do not have to reinstall torch + transformers.
